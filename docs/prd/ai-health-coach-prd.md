# AI 个人健康教练系统 — 产品需求文档

> **版本：** v0.1  
> **日期：** 2026-06-07  
> **状态：** 草稿  
> **来源：** 微信公众号文章《我朋友用 Claude 减掉了 20 公斤！没有健身房，没有应用，只有一个有效的系统！》— Thea

---

## 一、问题陈述

**现状问题：** 减肥和健康管理的最大障碍不是缺乏知识，而是缺乏个性化、持续性和低成本的专业指导。市场上的健身 APP 和减肥课程要么价格高昂、要么需要专属设备、要么缺乏针对个人情况的动态调整能力，导致用户难以坚持。

**目标用户：**
- 想减肥/维持健康体重但不想花钱请私教或买健身器材的人
- 有特定健康限制（过敏源、饮食禁忌、运动伤病）的人
- 尝试过减肥但缺乏持续动力和系统化方案的人

**不解决的代价：** 用户继续在碎片化的信息中迷失方向，尝试各种不可持续的方法，最终放弃并陷入健康焦虑。

---

## 二、目标

| # | 目标 | 衡量标准 |
|---|------|---------|
| 1 | 实现一次配置即可持续使用 | 用户填写一次个人信息，生成长期可迭代的方案 |
| 2 | 覆盖减重全链路要素 | 方案同时覆盖营养、运动、行为、睡眠、心理 5 个维度 |
| 3 | 零额外成本 | 用户无需买器械、补剂、付费 APP 或课程 |
| 4 | 可长期使用不疲乏 | 方案设计可持续 3 个月以上，而非短时冲刺 |
| 5 | 对 AI 新手可用 | 不要求 prompt engineering 知识，即开即用 |

---

## 三、非目标

| # | 范围外 | 原因 |
|---|--------|------|
| 1 | 实时健康数据采集（心率、血氧、步数） | 需要硬件支持，增加门槛 |
| 2 | AI 生成图像/视频健身示范 | 技术复杂且成本高 |
| 3 | 社交功能（排行榜、社区） | 偏离核心价值，增加隐私风险 |
| 4 | 医学诊断级建议 | AI 不具备医疗资质，需明确免责 |
| 5 | 定制化食材配送/电商对接 | 增加商业化复杂度，非 MVP 所需 |

## 四、用户场景

### 场景一：初始用户配置

**作为** 一个想减肥但无从下手的人，  
**我想要** 一次对话完成个人信息输入（体重、身高、年龄、目标、饮食限制、运动条件），  
**以便** 获得一套量身定制的完整方案。

### 场景二：渐进式营养方案

**作为** 一个按方案执行的用户，  
**我想要** 每天告诉我具体吃什么、吃多少、什么时间吃，  
**以便** 不需要自己花时间计算热量或搭配食谱。

### 场景三：无器材居家运动

**作为** 一个不想去健身房也没有器材的用户，  
**我想要** 一套只靠自重就能完成的每日训练计划，  
**以便** 在任何地方都能执行。

### 场景四：间歇性断食指引

**作为** 一个对间歇性断食感兴趣的用户，  
**我想要** 清晰的断食方案选择（16:8 / 5:2 / 24h）和每日执行指引，  
**以便** 安全有效地结合断食减脂。

### 场景五：动力维持

**作为** 一个容易在半途放弃的用户，  
**我想要** 心理支持话语、里程碑提醒和习惯追踪，  
**以便** 保持长期执行的动力。

### 场景六：进度调整与复盘

**作为** 一个执行了 2 周的用户，  
**我想要** 根据体重变化反馈调整方案（Plateau 突破、热量调整），  
**以便** 适应身体变化而非盲目执行固定方案。

## 五、功能需求

### P0 — 必做功能

| ID | 功能 | 描述 | 验收标准 |
|----|------|------|---------|
| F01 | 个人信息收集 | AI 引导用户输入：身高/体重/年龄/性别/目标体重/体脂率/饮食限制/过敏源/运动伤病/可用时间 | 一轮对话完成所有参数采集，输出结构化的用户画像摘要 |
| F02 | 个性化营养计划 | 根据 TDEE 计算每日热量缺口，分配三大宏量营养素比例，生成日食谱（3 餐 + 可选加餐） | 方案含具体食物选项，带分量估计（如 1 拳饭/1 掌肉），标注过敏原替换 |
| F03 | 居家无器械训练计划 | 基于用户运动能力水平，输出每周 3-5 天的自重训练安排（开合跳、俯卧撑、深蹲等变体） | 含组数/次数/休息间隔，提供动作文字指导，含渐进超负荷机制 |
| F04 | 16:8 间歇性断食方案 | 根据用户作息推荐具体进食窗口（如 12:00-20:00），说明断食期间可摄入物（水/黑咖啡/无糖茶）及注意事项 | 含非断食日的可替代方案，标注不适合人群 |

### P1 — 建议有

| ID | 功能 | 描述 | 验收标准 |
|----|------|------|---------|
| F05 | 购物清单生成 | 根据营养方案自动生成一次性采购清单，按超市品类分组（蔬果/蛋白/主食/调料） | 清单可直接带着去超市买，标注每项大约用量 |
| F06 | 日常动力教练 | 每日生成一条 motivational 消息（Quote / 小知识 / 鼓励） | 内容因人而异，不重复，有阶段性 |
| F07 | 水与睡眠优化 | 根据用户体重计算出每日目标饮水量，提供睡眠优化技巧（睡前习惯、环境调整） | 给出具体可执行的建议，如「睡前 1 小时停蓝光」 |
| F08 | 生活方式嵌入 | 将运动/饮食融入日常场景（如站着开会、走路上班、购物时多走路） | 建议与用户的实际生活场景匹配（办公/学生/居家） |

### P2 — 未来可做

| ID | 功能 | 描述 |
|----|------|------|
| F09 | 减重速度预测曲线 | 基于用户数据预测每周减重趋势，降低不切实际的预期 |
| F10 | 平台期突破策略 | 当体重连续 2 周无变化时，自动调整热量/运动方案 |
| F11 | 食谱多样性扩展 | 引入 cuisin rotation（中式 / 西式 / 亚洲风味交替） |
| F12 | 体脂率估算 | 基于腰围 / 体重变化估算体脂变化，而非仅看体重 |
| F13 | 跨语言支持 | 支持中英文双语方案生成 |

## 六、技术实现方案

### 架构概述

系统由 **7 个独立的 Prompt 模块**组成，每个模块作为 Claude Project 中的一个 Custom Instruction：

```
Claude Project
├── ① 个人营养计划 (Nutrition Coach)
├── ② 无器材居家锻炼 (Workout Coach)
├── ③ 间歇性断食指南 (Fasting Guide)
├── ④ 经济购物清单 (Budget Shopping)
├── ⑤ 日常动力教练 (Motivation Coach)
├── ⑥ 水和睡眠优化 (Hydration & Sleep)
└── ⑦ 生活方式减脂 (Lifestyle Integration)
```

### 核心配置

- **平台：** Claude.ai 的 Project 功能
- **方式：** Project 级别的 Custom Instructions
- **知识库：** 用户信息存放于 Project Knowledge（可上传个人数据文件）
- **交互模式：** 多轮对话，每日 Check-in + 按需查询

### 数据流

```
用户输入（体重/饮食日志/运动完成情况）
        ↓
Claude Project → 7 个 Custom Instructions 协同推理
        ↓
个性化输出（当日计划 / 调整建议 / 鼓励消息）
```

### 关键设计

- **状态管理：** 对话上下文在 Claude 的 Project 对话中持续保持，用户每次打开可以接着上次对话继续
- **渐进调整：** 通过追加指令让 AI 根据新数据更新原有方案
- **模块独立：** 7 个 prompt 解耦，任何一个模块可以单独更新而不影响其他
- **Zero-customization 入口：** 用户只需要回答基本问题，无需学习 prompt engineering

## 七、成功指标

| 指标 | 目标值 | 测量方式 |
|------|--------|---------|
| 首次配置完成率 | >80% | 用户一轮对话完成个人信息输入的比例 |
| 7 天留存 | >60% | 初次使用后第 7 天仍返回对话的用户比例 |
| 4 周减重达标率 | >70% 用户体重下降 ≥ 初始体重 2% | 用户上报的体重变化 |
| 用户满意度 | NPS ≥ 40 | 使用 2 周后的主观评价 |
| PRD 转化为实际知识库条目 | 100% (N=1) | Quant Wiki 收录该方案 |
